# Contributing

Contributions are welcome! Please read the following guidelines before submitting a pull request.

## Adding an item

- Make sure the item is related to the Flyte ecosystem.
- Search existing entries to avoid duplicates.
- Use the following format: `- [Name](url) - Description starting with uppercase, ending with a period.`
- Add the item at the end of the relevant section.
- Check your spelling and grammar.
- The item must not be unmaintained, archived, or deprecated.

## Pull request process

1. Fork the repository and create a branch from `main`.
2. Make your changes in the `readme.md` file.
3. Run `npx awesome-lint` to validate your changes.
4. Submit a pull request with a clear description of what you added and why it belongs.

## Quality standards

Every item should be something you have used or can vouch for. This is a curated list of the best resources, not a comprehensive directory.

## Updating your pull request

If your pull request is not accepted right away, you may be asked to make changes. There is no need to open a new pull request, just update the existing one.
